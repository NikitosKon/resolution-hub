# Resolution Hub — linkbuilding package

Статус: рабочий пакет для ручного запуска. Ничего не опубликовано и не отправлено автоматически.

## Состав

| Артефакт | Назначение | Следующее действие |
|---|---|---|
| `resource-page-plan.md` | 10 briefs для полезных resource-страниц с отдельным intent и источниками | выбрать 1–2 briefs и провести V4-пайплайн перед написанием |
| `research-briefs.md` | 2 идеи исследований с методологией и требованиями к данным | подтвердить доступ к данным, затем собрать датированный evidence pack |
| `outreach-templates.md` | 20 персонализируемых писем/углов для ручного outreach | выбрать релевантный тип получателя и персонализировать фактами страницы |
| `internal-linking-map.md` | карта связей для опубликованных PayPal, eBay и Grailed guides | внедрять ссылки только после проверки locale и URL |

## Порядок запуска

1. Подтвердить источники и данные для одного resource brief.
2. Запустить для него research → SERP → architecture → drafting → QA; не публиковать при блокирующих маркерах.
3. Опубликовать только после `DECISION: PASS`, технической проверки и отдельного разрешения.
4. После появления страницы обновить внутреннюю карту и только затем делать ручной outreach.
5. Измерять referring domains, qualified referral sessions, assisted conversions и качество ссылочного контекста; не считать количество писем KPI.

## Правила безопасности

- Не покупать ссылки, не использовать PBN, автоматические комментарии, массовые каталоги или скрытые редиректы.
- Не обещать редакторам эксклюзивность, позицию в Google или результат для их читателей.
- Не выдавать неподтверждённые цифры за исследование; неизвестные данные помечать `SOURCE_FRESHNESS_REQUIRED` или `MANUAL_REVIEW_REQUIRED`.
- Не отправлять письма без персонализации и проверки, что ресурс действительно связан с темой.
- Не публиковать preview, draft, admin и технические URL.

# Intent Analysis

1. **Primary intent — message triage.** The user already has, or suspects they have, an MC011 notice and needs to identify what the authenticated message asks before sending anything.
2. **Secondary intent — classify the affected function.** Searchers ask whether listings, selling, messages, payouts, or account access are affected. The page should ask the reader to record the exact blocked function instead of assuming every MC011 has the same effect.
3. **Secondary intent — distinguish request types.** Tracking/proof of delivery, identity, address, inventory/purchase evidence, outstanding buyer issues, and other actions appear in different examples. No sampled source supports treating them as one universal checklist.
4. **Safety intent — verify the message and protect data.** eBay help directs users to eBay Messages; readers also need a warning not to send passwords, authentication codes, full card numbers, or fabricated/altered evidence.
5. **Conditional intent — payouts and appeal.** Users want to know whether payouts are held and whether an appeal is available. Official pages make these case- and market-dependent; no universal time or outcome should be promised.
6. **Unmet Russian/Ukraine intent.** The returned evidence is mainly English/Japanese and not country-personalized. A Russian guide can translate the decision process, but must state the registered eBay market and keep local instructions separate.
